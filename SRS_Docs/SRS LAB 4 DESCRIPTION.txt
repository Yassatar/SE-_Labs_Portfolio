1. Sequence Diagram
 Description

This sequence diagram illustrates the interaction between system components during an exam session.

 Behavior
The student initiates the session by logging in.
The system validates credentials through the server.
An alternative flow (ALT) is used:
If authentication is successful → access is granted.
If authentication fails → an error is returned.
The student interacts with the system (answering questions).
The system processes the submission and calculates the result.
 Purpose

To show the chronological order of interactions between system components and highlight conditional flows.

 3. Activity Diagram

 Description

This activity diagram models the workflow of the exam process.


 Flow
Student logs in
System validates credentials

Decision point:

If valid → continue
If invalid → terminate process
Start exam
Answer questions
Auto-save responses (periodic process)
Submit exam

Decision point:

If submission is valid → calculate result
Otherwise → reject submission
Display final result

 Purpose

To illustrate the flow of activities and decision-making within the system.
